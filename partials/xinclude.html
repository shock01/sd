SD XInclude

ng:include does not work on svg elements because the browser does not parse the svg content as svg elements. HTML5 supposed to be namespace unware. But it isn't. So working on a replacement solution for ng:include

Bare example

<sd:xinclude href="includes/test.svg"/>

Rects first

<sd:xinclude href="includes/rects.svg" xpointer="//*[@id='first']"/>

Rects second

<sd:xinclude href="includes/rects.svg" xpointer="//*[@id='second']"/>
